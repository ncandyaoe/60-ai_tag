# 产品出海小标签AI自动生成需求文档

## 1. 项目背景与目标
随着公司出口的产品不断增加，标签设计的业务量急剧扩张。其中存在一类纯文字排版的标签（如国内进口产品上常见的黑白小标签）。为替代人工调整的繁琐过程，计划引入 AI 能力，打通内部系统（PLM）数据，实现该类小标签的自动排版与生成，要求最终做到**无需人工干预**，并直接输出可用于印刷的矢量型文件（如 Illustrator 支持的 .ai 格式等）。

## 2. 核心痛点与挑战
* **动态内容与固定尺寸的排版冲突**：标签尺寸是固定的模板（例如 3×3 大小），但不同国家或客户需要的文案长度差异巨大（如去美国为100个字，去英国为50个字）。AI 必须能按可用面积**自动适配字号、行距及排版**，解决字少时画面留白过空、字多时内容超出的问题。
* **各国合规性要求极为严格**：营养成分表等部分内容为强制合规模块，绝对不可修改或遮挡；各出口国对标签格式、字体、字号、**最小字符高度**均有法规设定的门槛要求。
* **业务场景高度碎片化**：需同时兼容不同语言、同一产品的不同口味配料差异、不同国家对图标（如环保标识）的不同诉求等。

## 3. 业务流程与交互
1. **源头数据录入**：在 PLM 系统界面的源头录入产品相关信息（包含净含量、中英文配料表与致敏原、营养成分表等）。
2. **多语言合成**：PLM 系统内部录入并匹配对应国家的翻译信息，自动合成产品信息表数据源。
3. **AI 自动排版引擎**：接收前端/系统传输的数字化内容及翻译文本，结合排版规则及法规约束，进行动态排版渲染。
4. **稿件输出**：系统最终输出带有完整排版的正式稿件（矢量型文件）。

## 4. 排版规则与业务场景要求

### 4.1 核心通用排版规则（简单标签约束）
1. **固定与统一**：采用基于单个固定尺寸的模板进行信息填充，并保证该模板内字段排版出现顺序始终一致。
2. **强制最低字高**：文字必须通过技术设定**锁死最低字高限制**。允许基于空间需要进行字体的横向或纵向变形，但绝不允许低于合规要求的最低字高。
3. **特定标记强调**：所有的冒号（`:`）前方字段，必须统一做**加粗**处理。
4. **格式允许断行**：在合规允许范围内，针对内容较长的情况，允许系统对其作换行（断行）处理。

### 4.2 场景扩展与复杂类型处理（复杂标签约束）
1. **单一标准多客户适配**（同一标准产品的不同细分）：
   * 产品的规格相同，但面向的客户/经销商不同（如进口商由A变成B），AI 需在模板内自动拉取并替换相关翻译信息，并保证周边排版不受破坏。
2. **同一标准扩展到不同产品**：
   * 面对产品衍生（如“生抽酱油”与“老抽酱油”），虽然标签体系相同，但其营养成分表（NRV%、Energy等数值）、配料及致敏原表述会存在差异，标签生成需全动态匹配这些参数而不出错。
3. **多语言混合展示**：
   * 一张固定尺寸的标签上，需要支持多种语言共存展现的复杂排版类型（如多国语言合并、小语种等）。
4. **动态图标与环保标识**：
   * 标签下方或部分固定位置，必需依靠具体单品信息的不同，自动带出并排列不同的环保类标识或要求图标（例如：加拿大要求包含放大镜图标及指定格式的营养成分表格）。
5. **基于国家属性的独有样式**：
   * 必须兼容包含但不限于澳大利亚、马来西亚、俄罗斯等多国的独特样式表述（涵盖特定的成分强调、营养表隔离框及特有的图文混排模式）。
